# Copilot Instructions — PC Media Center

## Проект
Десктопный полноэкранный медиацентр/лаунчер на Flutter (Windows, Linux, macOS).

## Архитектура
Читай `ARCHITECTURE.md` перед любой задачей. Проект использует **Clean Architecture**:
- `domain/` — entities, repository interfaces, use cases (чистый Dart, без зависимостей)
- `data/` — repository implementations, datasources, models, database (Drift)
- `presentation/` — BLoC, screens, widgets

## Правила кода

### Общие
- Язык кода и комментариев: **английский**
- Язык UI-строк: через **i18n** (`.arb` файлы), не хардкодить текст
- State management: **flutter_bloc** (BLoC pattern)
- DI: **get_it** + **injectable**
- Routing: **go_router**
- Database: **drift** (SQLite)

### Структура BLoC
Каждый BLoC состоит из 3 файлов:
- `*_bloc.dart` — логика
- `*_event.dart` — события
- `*_state.dart` — состояния

Используй `Equatable` для events и states. Используй `sealed class` для states если Dart 3+.

### Entities vs Models
- **Entity** (`domain/entities/`) — бизнес-объект, без `fromJson`/`toJson`
- **Model** (`data/models/`) — DTO с сериализацией, маппится в Entity

### Use Cases
- Один Use Case = одна операция
- Метод `call()` для удобного вызова

### Виджеты
- Переиспользуемые виджеты в `presentation/widgets/`
- Экраны в `presentation/screens/<feature>/`
- Виджеты НЕ обращаются к BLoC напрямую — получают данные через параметры

### Навигация фокусом
- Все интерактивные элементы оборачивать в `FocusableWidget`
- Поддерживать навигацию стрелками / Tab

### База данных
- Файл БД хранится рядом с исполняемым файлом: `data/media_center.db`
- Миграции через Drift

### Тестирование
- `mocktail` для моков
- `bloc_test` для тестов BLoC
