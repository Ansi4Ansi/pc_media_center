## State Management

### Primary: BLoC Pattern
- **Package**: `flutter_bloc` v9.1.0
- **Architecture**: CQRS-inspired pattern with events, states, and blocs
- **Benefits**:
  - Predictable state transitions
  - Easy testing (deterministic behavior)
  - Clear separation of concerns
  - Excellent AI-agent friendliness

### Supporting Libraries
- `equatable` v2.0.7 — Value equality for events/states
- `bloc_test` v10.0.0 — BLoC-specific unit testing

## Dependency Injection

| Tool | Purpose |
|------|--------|
| **get_it** v8.0.3 | Simple service locator with lazy initialization |
| **injectable** v2.5.0 | Compile-time DI generation for cleaner code |

### DI Pattern
```
domain/     → Interfaces only (no dependencies)
data/      → Implementations injected via get_it
presentation → Consumes interfaces from domain
```

## Technology Decisions Summary

1. **Flutter** — Single codebase for Windows/Linux/macOS with native performance
2. **Drift over sqflite** — Type-safe SQL, better migrations, portable storage
3. **BLoC pattern** — Predictable state management, easy to test and debug
4. **go_router** — Modern declarative routing with deep link support
5. **get_it + injectable** — Clean DI without framework-specific solutions
6. **Offline-first design** — Works without internet, caches metadata locally
