## Overview

PC Media Center follows Clean Architecture principles with strict dependency rules.

## Architectural Layers

```
┌─────────────────────────────────────────────────┐
│                 Presentation                     │
│  Screens → BLoC → Widgets                       │
├─────────────────────────────────────────────────┤
│                   Domain                         │
│  Entities, Use Cases, Repository Interfaces      │
├─────────────────────────────────────────────────┤
│                    Data                          │
│  Repository Impl, DataSources (Local + Remote)   │
│  Database (Drift/SQLite), API clients            │
└─────────────────────────────────────────────────┘
```

### Dependency Rule
- Presentation depends on Domain only
- Data depends on Domain only
- Domain has no dependencies (pure Dart classes)

## Database Schema

### Categories Table
- id: INTEGER PRIMARY KEY AUTOINCREMENT
- name: TEXT NOT NULL
- icon: TEXT (icon filename or path)
- sort_order: INTEGER DEFAULT 0
- is_movie_type: BOOLEAN DEFAULT FALSE
- scan_paths: TEXT (JSON array of media paths)
- file_extensions: TEXT (allowed extensions)
- created_at/updated_at: DATETIME

### Items Table
- id: INTEGER PRIMARY KEY AUTOINCREMENT
- category_id: INTEGER REFERENCES categories(id) ON DELETE CASCADE
- title: TEXT NOT NULL
- description: TEXT
- poster_path/url: TEXT (local path and URL)
- launch_path/args: TEXT (executable and arguments)
- item_type: TEXT DEFAULT 'file' ('app'|'file'|'media'|'url')
- year, rating, external_id: TEXT/REAL
- metadata_json: TEXT (additional metadata)
- sort_order, is_favorite: INTEGER/BOOLEAN
- created_at/updated_at: DATETIME

### Settings Table
- key: TEXT PRIMARY KEY
- value: TEXT NOT NULL

## API Integration Architecture

### TMDB (TheMovieDB)
- Endpoint: https://api.themoviedb.org/3/
- Purpose: Movie search, metadata, posters
- Auth: User-provided API key in settings
- Caching: Results stored in SQLite

### Kinopoisk (Unofficial)
- Endpoint: https://kinopoiskapiunofficial.tech/api/
- Purpose: Russian movie database, ratings
- Auth: User-provided API key in settings

## Icon Extraction Architecture

### Platform-Specific Implementation

**Windows:** ExtractIcon/SHGetFileInfo via FFI (win32 package)
**macOS:** NSWorkspace.icon() via Method Channel or sips CLI
**Linux:** Parse .desktop file + search /usr/share/icons/

Fallback: Use placeholder icon if extraction fails.

---

## Offline Metadata Extraction

### Filename Parsing
- Extract title and year from filenames
- Remove quality tags (BDRip, 1080p, etc.)
- Examples:
  - "Matrix (1999) BDRip.mkv" → title="Matrix", year=1999
  - "halflife2.exe" → title="halflife2"

### Metadata Sources Priority
1. File metadata (ID3, ffprobe, IconExtractor)
2. Filename parsing fallback
3. Online search enrichment (if internet available)
