## lib/data/ — Data Layer Implementation

### data/database/
| File | Purpose |
|------|--------|
| `app_database.dart` | Drift database class definition and initialization |
| `tables/categories_table.dart` | Categories table schema definition |
| `tables/items_table.dart` | Items table schema definition |
| `daos/categories_dao.dart` | Category data access object (CRUD operations) |
| `daos/items_dao.dart` | Item data access object (CRUD operations) |

### Database Initialization Flow
```
1. App starts → main.dart
2. configureDependencies() called
3. Database initialized with migrations
4. DAOs created and registered in DI
5. Repositories use DAOs for database operations
```

## Summary of Layer Responsibilities

```
┌─────────────────────────────────────────────────┐
│              Presentation (UI)                   │
│  Screens, Widgets, BLoCs — User interaction     │
├─────────────────────────────────────────────────┤
│                 Domain (Business Logic)           │
│  Entities, Use Cases, Repository Interfaces     │
├─────────────────────────────────────────────────┤
│                  Data (Persistence)               │
│  Repositories, DAOs, DataSources, Models         │
└─────────────────────────────────────────────────┘
```

Dependency Flow: Presentation → Domain ← Data
