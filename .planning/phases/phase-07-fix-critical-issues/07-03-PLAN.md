  <action>
    Fix the improper BLoC lifecycle by registering ItemBloc as factory (not singleton):
    
    PROBLEM: ItemBloc is registered as singleton but creates new instance via getIt in screen.
    This causes state management issues and conflicts between screens.
    
    SOLUTION: Register ItemBloc as factory and use BlocProvider:
    
    1. Open lib/core/di/injection.dart
    2. Find ItemBloc registration (search for "ItemBloc")
    3. Change from singleton to factory:
       
       Before (WRONG):
       ```dart
       getIt.registerLazySingleton<ItemBloc>(() => ItemBloc(...));
       // or
       getIt.registerSingleton<ItemBloc>(ItemBloc(...));
       ```
       
       After (CORRECT):
       ```dart
       getIt.registerFactory<ItemBloc>(() => ItemBloc(
         getIt<GetItemsByCategory>(),
         getIt<AddItem>(),
         // ... other dependencies
       ));
       ```
    4. Open lib/presentation/screens/category/category_screen.dart
    5. Find where ItemBloc is created (around line 40, likely `final bloc = getIt<ItemBloc>()`)
    6. Replace with BlocProvider pattern:
       
       Before (WRONG):
       ```dart
       @override
       void initState() {
         super.initState();
         final bloc = getIt<ItemBloc>(); // Creates new instance but singleton causes issues
         // ...
       }
       ```
       
       After (CORRECT):
       ```dart
       // In build() method, wrap content with BlocProvider:
       @override
       Widget build(BuildContext context) {
         return BlocProvider(
           create: (context) => getIt<ItemBloc>(), // Factory creates fresh instance
           child: Builder(
             builder: (context) {
               final bloc = context.read<ItemBloc>();
               // ... rest of build
             }
           ),
         );
       }
       ```
    7. Or if already using BlocProvider, ensure it's using create: not value:
       ```dart
       // CORRECT - creates new instance
       BlocProvider(create: (context) => getIt<ItemBloc>())
       
       // WRONG - uses existing (problematic)
       BlocProvider.value(value: getIt<ItemBloc>())
       ```
    8. Verify with flutter analyze
    
    Key changes:
    - injection.dart: registerFactory<ItemBloc> instead of singleton
    - category_screen.dart: Use BlocProvider with create: parameter
    - Each screen gets its own ItemBloc instance
  </action>
