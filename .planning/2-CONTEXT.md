## 🏗️ Архитектурные решения

### 1. Добавление категории — Диалоговое окно

**Решение:** Использовать `showDialog` с текстовым полем и кнопками.

**Обоснование:**
- Минимальное количество экранов
- Быстрый доступ к функционалу
- Не требует навигации
- Подходит для простых форм ввода

**Реализация:**
```dart
// В CategoryBloc или HomeScreen
showDialog(
  context: context,
  builder: (context) => AlertDialog(
    title: Text('Новая категория'),
    content: TextField(...),
    actions: [...],
  ),
);
```

---

### 2. Редактирование/удаление категории — Отдельный экран

**Решение:** Создать отдельный экран `CategoryEditScreen`.

**Обоснование:**
- Возможность отображения дополнительных информации (иконка, описание)
- Удобство работы с формой редактирования
- Возможность валидации перед сохранением
- Поддержка режима "только просмотр" для чтения

**Структура экрана:**
```dart
CategoryEditScreen(
  category: categoryOrNull,
  mode: EditMode.editOrView, // edit | view
)
```

---

### 3. Переход в список элементов — Ленивая загрузка

**Решение:** Бесконечная ленивая загрузка с начальной пачкой из 50 элементов.

**Обоснование:**
- **Начальная загрузка 50 элементов**: Быстрый отклик, меньше данных при старте
- **Ленивая загрузка**: По мере скролла вниз — оптимальный баланс между производительностью и памятью
- **Кнопка "Обновить"**: Для ручного обновления списка без перезагрузки страницы

**Параметры:**
| Параметр | Значение |
|----------|----------|
| Начальная пачка | 50 элементов |
| Размер пачки при скролле | 50 элементов |
| Порог срабатывания | Когда последний элемент виден на экране |

**Реализация:**
```dart
// В CategoryScreen
final items = useState<ItemEntity[]>([]);
final isLoading = useState(false);
final hasMore = useState(true);

// Начальная загрузка
await _loadItems(0, 50);

// Ленивая загрузка при скролле
ScrollController.addListener(() {
  if (hasMore && !isLoading) {
    // Проверка порога срабатывания...
    _loadItems(currentCount + 50);
  }
});
```

## 🎨 UI/UX решения

### 1. Диалог добавления категории

**Компоненты:**
- `TextField` для ввода названия
- Кнопка "Отмена" (левая)
- Кнопка "Добавить" (правая)

**Поведение:**
- Валидация: название не должно быть пустым или дубликатом
- При вводе — автозаполнение иконок из категории (если есть)

---

### 2. Экран редактирования категории

**Режимы работы:**
| Режим | Описание |
|-------|----------|
| `edit` | Редактирование названия, выбор иконки |
| `view` | Только просмотр информации (для чтения) |

**Элементы интерфейса:**
- Иконка категории (изображение или цветной квадрат)
- Название категории (текстовое поле в режиме редактирования)
- Кнопка "Сохранить" / "Отмена"
- Кнопка удаления (в режиме редактирования)

---

### 3. Список элементов с ленивой загрузкой

**Визуальные индикаторы:**
- Скелетон во время загрузки новой пачки
- Индикатор "Загрузка..." при достижении порога
- Кнопка "Обновить" для ручного обновления списка

**Поведение:**
- Плавная анимация перехода между пачками
- Визуальная обратная связь о загрузке новой порции данных
- Сохранение позиции скролла при переключении категорий

## ⚠️ Ограничения

### 1. Производительность при большой коллекции элементов

**Ограничение:** При очень большом количестве элементов (>1000) может потребоваться:
- Виртуализация списка (`ListView.builder`)
- Кэширование изображений постеров
- Ленивая загрузка только видимых элементов

### 2. Память при бесконечной загрузке

**Решение:**
- Очистка невидимых элементов из памяти
- Кэширование изображений с LRU (см. фазу 12)
